### 3. 评分归一化方法

所有评分归一化到 [1, 5] 区间，与模型原有接口兼容。为处理杭州等大城市的数据碾压效应，对高度偏态的指标使用 log 变换后再做 min-max 归一化。

| 评分字段 | 原始指标 | 变换方法 |
|---|---|---|
| `population_score` | 年末常住人口（万人） | log1p → minmax [1,5] |
| `gdp_score` | 生产总值（亿元） | log1p → minmax [1,5] |
| `transport_score` | 综合交通便利度 | 各分量 log1p 后加权求和 → minmax [1,5] |
| `football_score` | 体育场地数绝对值(70%) + 人均密度(30%) | log1p → minmax [1,5]；体育氛围代理 |
| `sports_base_score` | 体育场地数（个） | log1p → minmax [1,5]；体育基础代理 |

**transport_score 加权公式**：

```
score = log1p(road_km) × 0.25
      + log1p(expressway_km) × 0.20
      + log1p(passenger_volume_10k) × 0.20
      + log1p(rail_passenger_10k) × 0.20
      + log1p(auto_count) × 0.15
```

各分量使用 log1p 保证不同量级指标的可比性（如公路里程数千公里 vs 轨道交通 0~2000 万人次）。

### 保持兼容的接口

- `scripts/run_problem34.py` 的运行方式保持不变，仍统一读取 `data/raw/region_metrics.csv` 并输出 `results/problem34/`。
- `src/problem34/venue_model.py` 的总评分框架保持不变，但 `capacity_score` 已改为由体育基础、具体场馆容量、铁路可达性和机场可达性构成。
- `src/problem34/weights.py` 集中保存对照情景权重，并提供熵权法函数。当前主方案使用熵权法动态计算赛事影响、承办能力和总目标三层权重。

### 承办能力评分更新

`capacity_score` 已从“交通 + 足球氛围代理”更新为：

```text
capacity_score =
    alpha1 * sports_base_component_score
  + alpha2 * stadium_capacity_score
  + alpha3 * rail_access_score
  + alpha4 * airport_access_score
```

其中 `sports_base_component_score` 由 `sports_base_score` 换算到 0-100 分，只作为体育基础辅助项；`alpha1` 至 `alpha4` 由 64 个地区的承办能力子指标矩阵通过熵权法计算得到。

缺失和不完整数据采用保守分层处理：

- `expressway_km`、`passenger_volume_10k`、`rail_transit_passenger_10k`、`auto_count` 为空时，在交通综合分中按 0 处理。其中 `rail_transit_passenger_10k` 为空只表示年鉴该统计项为空，不等同于无铁路客运站。
- `stadium_capacity` 或 `stadium_name` 为空：容量分取最低档 20 分。
- `data_quality` 含 `partial`：容量分乘以 0.85。
- `data_quality` 含 `verified(在建)`：容量分乘以 0.75。
- `has_rail_station` 含 `无铁路客运站`：铁路分 0；含 `最近` 或 `客运有限`：铁路分 60；其他明确铁路站：铁路分 100。
- `nearest_airport_km` 按距离分段计分；机场名称含 `在建` 时机场分再乘以 0.7。

## 已知数据缺口

| 缺口 | 影响 | 建议补充来源 |
|---|---|---|
| 杭州/宁波/绍兴城市轨道交通客运量 | 年鉴分区县表不含地铁数据，transport_score 可能被低估 | 各市 2024 年统计公报或轨道交通年报 |
| 具体场馆容量 | 10 个地区 `stadium_capacity` 仍为空，1 个地区 `stadium_name` 为空 | 继续查地方体育局官网、体育中心公开信息 |
| 人工补充字段质量 | 部分铁路、机场、场馆字段为 `partial` 或 `verified(在建)` | 论文中需作为人工公开资料补充项说明 |
| 部分城区或海岛县的高速公路里程、客运量、轨道交通客运量、民用汽车拥有量为空 | 当前清洗脚本按 0 参与交通评分，可能低估个别地区 | 后续可改为按可用分量重加权，或查地方交通统计补齐 |

## 使用方式

```bash
# 重新清洗数据（如需更新年鉴文件后重跑）
python3 scripts/clean_problem3_manual_sources.py

# 仅检查年鉴文件
python3 scripts/clean_problem3_manual_sources.py --inspect

# 干跑验证映射（不写输出）
python3 scripts/clean_problem3_manual_sources.py --dry-run

# 运行完整模型
python3 scripts/run_problem34.py
```

输出文件均在 `results/problem34/` 目录下。
