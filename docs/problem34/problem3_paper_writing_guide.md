# 问题 3 论文写作要点

## 1. 论文主线

问题 3 的任务不是重新分组，而是在问题 1 已经给出 16 个小组的基础上，选择 8 个比赛地点，并确定每个地点承办哪两个小组。因此论文叙述应突出“承接问题 1 分组结果，进一步建立多目标离散设施选址模型，解决赛点选址与小组合并承办问题”。

推荐模型命名：

> 带小组配对约束的多目标离散设施选址模型

这个命名比“综合评分模型”更贴合数学建模语言。64 个参赛地区是候选设施点，16 个小组是待服务对象，但服务单元不是单个小组，而是两个小组合并后的赛区。模型同时决定小组如何两两配对、哪些候选地被选中、每个小组对分配给哪个赛点。

建议主线写成：

1. 输入问题 1 的最终分组结果和 64 个地区的基础数据。
2. 构造出行便利、公平性、赛事影响、承办能力四类评价指标。
3. 将赛点选择建模为多目标离散设施选址问题，建立“小组对-候选举办地”的效用函数。
4. 使用熵权法根据候选方案矩阵自动确定权重。
5. 对所有“小组对-候选举办地”计算综合效用得分。
6. 使用 beam search 在组合空间中搜索 8 个互不重复赛点和 8 组小组配对。
7. 用三种对照权重情景和台风图检验主方案的敏感性。

## 2. 输入数据说明

### 2.1 继承组员 A 的分组结果

问题 3 读取 `results/problem1/best_groups.csv`，该文件是组员 A 在问题 1 中得到的最终分组结果。选择这组数据的原因有三点：

- 赛题逻辑上，问题 3 是在问题 1 分组之后安排比赛地点，不能脱离前一问另行生成小组。
- 组员 A 的结果已经综合考虑县市队分布、公平性和实力均衡，是问题 3 的上游约束。
- 直接继承问题 1 输出可以保证整篇论文从“分组”到“选址”的决策链条一致，避免不同问题之间使用不一致的数据口径。

输入字段包括 `group`、`team_name`、`level`、`parent_city`、`strength`。其中 `group` 决定小组归属，`team_name` 用于关联地区地理坐标和承办能力数据。

### 2.2 地区基础数据

问题 3 同时读取 `data/raw/region_metrics.csv`。该文件覆盖 64 个参赛地区，提供：

- 地理坐标：`lat`、`lon`，用于 Haversine 球面距离。
- 影响力指标：人口、GDP、交通、体育氛围相关分数。
- 承办能力指标：体育基础、代表性场馆容量、铁路可达性、机场可达性。
- 数据质量字段：用于说明人工补充场馆、机场、铁路信息的不确定性。

## 3. Pipeline 写法

### Step 1：读取并关联数据

算法：CSV 数据读取和键值关联。

输入：

- `best_groups.csv`：问题 1 的最终 16 个小组。
- `region_metrics.csv`：64 个地区的坐标、指标和场馆数据。

输出：

- `groups`：每个小组对应 4 支队伍。
- `regions_by_team`：每支队伍对应的地区指标。
- `venues`：64 个候选举办地。

### Step 2：建立离散设施选址框架

算法/模型：多目标离散设施选址模型。

集合定义：

- `G`：16 个小组。
- `V`：64 个候选举办地，也就是候选设施点。
- `P`：所有可行小组对，由 16 个小组两两组合得到。

决策内容：

- 从 `P` 中选择 8 个互不重叠的小组对，使每个小组恰好出现一次。
- 从 `V` 中选择 8 个互不重复的举办地。
- 将每个选中的小组对分配给一个选中的举办地。

约束：

- 每个小组只能被分配一次。
- 每个举办地最多使用一次。
- 每个举办地承办两个小组，即一个小组对。
- 候选举办地必须来自 64 个真实地区，不能产生连续空间中的虚拟中心点。

优化目标：

```text
max sum total_score(pair_k, venue_k)
```

这里的 `total_score` 是多目标综合效用，而不是单一距离目标。

### Step 3：构造距离指标

算法：Haversine 球面距离公式。

对任意小组对和候选举办地，计算 8 支队伍到举办地的距离，得到：

- `avg_distance_km`：平均出行距离。
- `max_distance_km`：最远出行距离。
- `distance_std_km`：出行距离标准差。

这些距离进一步转化为：

- `avg_travel_score`：平均出行便利度。
- `worst_travel_score`：最远出行控制得分。
- `fairness_score`：距离离散度和同母城市集中度的综合公平性得分。

写作时可以强调：仅最小化总旅行距离可能会牺牲个别远端队伍，因此本文同时使用平均距离和最远距离。平均距离控制总体出行成本，最远距离控制极端出行负担。

### Step 4：构造影响力和承办能力指标

算法：归一化指标的加权合成，权重由熵权法确定。

`impact_score` 使用人口、GDP、交通、体育氛围四个子指标。`capacity_score` 使用体育基础、场馆容量、铁路可达性、机场可达性四个子指标。

这里要强调：体育场地数是广义体育设施口径，不直接等同于大型比赛场馆；因此承办能力中单独引入代表性场馆容量，并对缺失、部分核实、在建场馆作保守处理。

写作时也要强调：最大化赛事影响不能只看覆盖人口，否则模型会天然偏向核心城市。本文将人口、GDP、交通和体育氛围纳入 `impact_score`，但同时保留出行公平和承办能力目标。

### Step 5：熵权法确定权重

算法：Entropy Weight Method。

输入：

- 64 个地区的影响力子指标矩阵。
- 64 个地区的承办能力子指标矩阵。
- 所有“小组对-候选举办地”的五目标得分矩阵。

输出：

- `impact_score` 内部权重。
- `capacity_score` 内部权重。
- 总目标权重：`avg_travel_score`、`worst_travel_score`、`fairness_score`、`impact_score`、`capacity_score`。

写作解释：

没有历史办赛效果标签，不能把权重写成回归系数。熵权法利用指标离散度确定权重，离散度越高，信息效用越大，越能区分候选方案，因此权重越高。当前结果中 `worst_travel_score` 权重最高，说明本题候选方案之间最主要的差异来自最远出行负担。

### Step 6：计算候选“小组对-赛点”效用

算法：设施选址效用函数。

输入：

- 一个小组对。
- 一个候选举办地。
- 熵权法得到的权重。

输出：

- 该“小组对-赛点”的 `total_score`，即设施选址模型中的边权或效用值。
- 同时保留平均距离、最远距离、公平性、影响力、承办能力、主场惩罚等解释字段。

评分公式：

```text
total_score =
    w1 * avg_travel_score
  + w2 * worst_travel_score
  + w3 * fairness_score
  + w4 * impact_score
  + w5 * capacity_score
  - home_penalty
```

### Step 7：Beam search 联合优化

算法：受控 beam search。

为什么不用简单贪心：

每个赛点要承办两个小组，且 8 个赛点不能重复。如果只逐个选择当前最优的小组对和赛点，可能提前占用某个关键城市，导致后续小组只能选择较差方案。因此需要联合搜索。

状态定义：

- `total_score`：当前已选方案总分。
- `remaining_groups`：尚未配对的小组。
- `used_venues`：已经使用的举办地。
- `plan`：当前已确定的小组对和赛点安排。

扩展方式：

1. 取剩余小组中的第一个小组。
2. 枚举它与其他剩余小组组成的小组对。
3. 对每个小组对尝试其前若干个候选赛点。
4. 跳过已经使用过的赛点。
5. 形成新状态，并按累计总分排序。
6. 每一层只保留前 `beam_width` 个状态，控制组合爆炸。

输出：

- 8 个小组对。
- 8 个互不重复的举办地。
- 总效用较高且满足约束的完整承办方案。

### Step 8：敏感性分析和台风图

算法：情景对比 + tornado chart。

四种方案：

- `entropy`：熵权主方案。
- `balanced`：综合均衡的专家权重对照方案。
- `travel_first`：出行优先对照方案。
- `impact_first`：影响力优先对照方案。

台风图的作用：

台风图适合展示“不同情景相对于主方案的指标偏移”。建议以 `entropy` 为基线，横向条形展示其他三种方案在平均距离、最大距离、平均公平性、总影响力上的百分比变化。采用百分比变化而不是绝对差值，是因为距离、得分和总影响力不在同一量纲上。它能增强直观性，因为读者可以一眼看到：熵权主方案主要压低最大出行距离，而影响力优先方案显著提高总影响力但牺牲出行便利。

注意：不同情景使用不同权重，`total_assignment_score` 不是同一把尺子下的效用值，论文中不建议用台风图比较总分。应该比较物理含义一致的指标，例如距离、公平性和影响力。

## 4. 和其他建模思路的比较

### 4.1 为什么不是单纯最小总旅行距离

单纯最小化所有队伍总旅行距离虽然直观，但会把所有队伍的距离简单相加，可能出现“多数队伍受益、少数远端队伍被明显牺牲”的方案。小组赛选址不仅要降低总体出行成本，也要控制最远队伍的负担。因此本文同时设置 `avg_travel_score` 和 `worst_travel_score`，前者对应总体出行效率，后者对应极端负担控制。

### 4.2 为什么不是单纯最大化覆盖人口或赛事影响

如果只最大化赛事影响，模型会明显偏向杭州、宁波、温州、金华、义乌等人口和经济基础较强的地区。这类方案传播效果较好，但不一定适合具体两个小组的出行结构。本文把赛事影响作为目标之一，而不是唯一目标，避免大城市偏置压倒参赛便利和公平性。

### 4.3 为什么不以 K-means 聚类为主模型

K-means 适合连续空间聚类，而本题举办地必须从 64 个真实候选地区中选择。K-means 产生的聚类中心可能落在没有球队、没有场馆或没有承办条件的位置，还需要再投影到最近地区，解释链条会变长。

更关键的是，K-means 很难自然处理以下约束：

- 两个小组合并为一个承办单元。
- 每个小组只能出现一次。
- 每个赛点最多使用一次。
- 场馆容量、铁路可达性、机场可达性进入目标函数。
- 主场优势需要软惩罚。

因此，K-means 可以作为探索性启发式，但不适合作为主模型。多目标离散设施选址模型更贴合题目结构。

## 5. 结果解读要点

主方案选出的赛点不是简单按城市规模排序。这个现象要主动解释：

- 赛点服务的是两个具体小组，而不是全省抽象排名。
- 模型是带小组配对约束的设施选址问题，目标是为具体小组对选择合适设施点，而不是给城市综合实力排位。
- 熵权法识别出最远出行负担在候选方案之间差异最大，因此主方案更重视避免个别球队被明显牺牲。
- 县市赛点进入最终方案并不表示其综合实力超过杭州、宁波等大城市，而是它们对对应小组的地理位置更均衡。
- 与 `balanced` 和 `impact_first` 相比，`entropy` 方案降低了最大出行距离，但牺牲了一部分总影响力，这是“出行公平”和“赛事传播”的可解释权衡。
