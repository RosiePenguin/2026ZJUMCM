# 问题 3：小组赛比赛地点选择模型说明

## 1. 建模目标

根据问题 1 的最终分组结果，选择 8 个市或县作为小组赛比赛地点，每个地点承办 2 个小组。本文将该任务建模为带小组配对约束的多目标离散设施选址问题：64 个参赛地区是候选设施点，16 个小组需要两两配对后分配给 8 个互不重复的设施点，优化目标同时考虑出行便利、公平性、赛事影响和承办能力。

模型同时考虑三类目标：

- 便利各队参赛：降低平均旅行距离和最大旅行距离。
- 保证公平性：避免个别队伍承担过高出行负担，并对主场优势进行软惩罚。
- 扩大赛事影响：优先选择人口、经济、交通和足球氛围综合条件较好的地区。

## 2. 数据与指标

基础数据文件为 `data/raw/region_metrics.csv`，覆盖 64 个参赛地区。当前数据已由 2025 年浙江统计年鉴的 2024 年县市表清洗得到，核心字段包括：

- `lat, lon`：地区中心点坐标，用于计算球面距离。
- `population_10k, gdp_100m`：常住人口和生产总值原始统计指标。
- `road_km, expressway_km, passenger_volume_10k, rail_transit_passenger_10k, auto_count`：交通相关原始统计指标。
- `sports_venue_count`：体育场地数原始统计指标。该口径为广义体育场地/设施点数量，可反映体育基础和群众体育氛围，但不等同于大型比赛场馆数量。
- `has_rail_station, nearest_airport, nearest_airport_km`：人工补充的铁路客运站、最近机场和距离信息。
- `stadium_name, stadium_capacity`：人工补充的代表性场馆名称和容量信息。
- `population_score`：由常住人口经 log-minmax 归一化得到，1-5 分。
- `gdp_score`：由生产总值经 log-minmax 归一化得到，1-5 分。
- `transport_score`：由公路里程、高速公路里程、客运量、轨道交通客运量和民用汽车拥有量加权归一化得到，1-5 分。
- `football_score`：由体育场地数绝对规模和人均体育场地密度加权归一化得到，作为体育氛围代理指标，1-5 分。
- `sports_base_score`：由体育场地数经 log-minmax 归一化得到，作为体育基础代理指标，1-5 分。

核心统计指标为官方年鉴数据，数据质量为 A 级。铁路客运站、机场距离、具体体育场名称和容量已由人工检索结果补入，但其中部分字段为 `partial` 或 `verified(在建)`，论文中应作为公开资料补充项单独说明。当前模型已将场馆容量、铁路可达性和机场可达性纳入承办能力评分；缺失和不完整数据采用保守分层处理。

## 3. 多目标设施选址模型

设 16 个小组集合为 `G`，64 个候选举办地集合为 `V`。模型需要同时决定：

- 小组配对：将 16 个小组划分为 8 个互不重叠的小组对。
- 赛点选择：从 `V` 中选择 8 个互不重复的举办地。
- 赛点分配：为每个小组对指定一个举办地。

因此，该问题不是简单的城市排序，而是一个带配对约束、唯一性约束和多目标效用函数的离散设施选址问题。对任意两个小组和一个候选举办地，先计算 8 支队伍到举办地的 Haversine 球面距离，再将该“小组对-赛点”组合转化为综合效用得分：

```text
total_score =
    w1 * avg_travel_score
  + w2 * worst_travel_score
  + w3 * fairness_score
  + w4 * impact_score
  + w5 * capacity_score
  - home_penalty
```

其中：

- `avg_travel_score`：由 8 支队伍平均旅行距离和举办地交通分共同决定。
- `worst_travel_score`：由最远队伍旅行距离决定，用于控制极端出行负担。
- `fairness_score`：综合距离离散度和同一母城市队伍集中度。
- `impact_score`：人口、GDP、交通、体育氛围按熵权法确定内部权重。
- `capacity_score`：由体育基础、具体场馆容量、铁路可达性和机场可达性综合得到，内部权重同样由熵权法确定；其中体育基础只作辅助项，避免把广义体育场地数误当成大型比赛场馆数。
- `home_penalty`：若举办地本队在承办的两个小组内，则扣分，体现主场优势软约束。

相应的优化目标可写为在满足每个小组恰好被分配一次、每个赛点最多使用一次的约束下，最大化 8 个“小组对-赛点”组合的综合效用之和：

```text
max sum total_score(pair_k, venue_k)
s.t. each group is assigned to exactly one pair
     each selected venue is used by at most one pair
     exactly 8 group pairs and 8 venues are selected
```

若只最小化所有队伍总旅行距离，模型可能忽略最远队伍的极端出行负担；若只最大化覆盖人口或赛事影响，方案容易集中到杭州、宁波、温州等大城市而弱化具体小组的地理公平性。因此本文采用多目标设施选址框架，将平均出行、最远出行、公平性、影响力和承办能力同时纳入。

承办能力评分为：

```text
capacity_score =
    alpha1 * sports_base_component_score
  + alpha2 * stadium_capacity_score
  + alpha3 * rail_access_score
  + alpha4 * airport_access_score
```

其中：

- `sports_base_component_score`：由 `sports_base_score` 从 1-5 分换算到 0-100 分，仅代表群众体育设施基础。
- `stadium_capacity_score`：由 `stadium_capacity` 经 log-minmax 归一化到 20-100 分；若容量或场馆名缺失，则取最低档 20 分。
- `rail_access_score`：无铁路客运站为 0 分；仅最近站或客运有限为 60 分；有明确本地客运站为 100 分。
- `airport_access_score`：按最近机场距离分段计分，30 km 内为 100 分，30-60 km 线性降至 80 分，60-100 km 线性降至 50 分，100-160 km 线性降至 20 分，超过 160 km 保持 20 分；机场在建时再乘以 0.7。
- `data_quality` 含 `partial` 时，场馆容量分乘以 0.85；含 `verified(在建)` 时，场馆容量分乘以 0.75。

## 4. 权重设定依据

由于没有历史办赛效果、票房收入或参赛满意度等标签数据，本文不把权重解释为统计回归系数。为减少专家赋权的主观性，主方案采用熵权法确定三层权重：

1. 对人口、GDP、交通、体育氛围 4 个子指标，在 64 个地区构成的指标矩阵上计算 `impact_score` 内部权重。
2. 对体育基础、场馆容量、铁路可达性、机场可达性 4 个子指标，在 64 个地区构成的指标矩阵上计算 `capacity_score` 内部权重。
3. 对 `avg_travel_score`、`worst_travel_score`、`fairness_score`、`impact_score`、`capacity_score` 5 个目标指标，在所有“小组对-候选举办地”组合构成的决策矩阵上计算总目标权重。

熵权法的核心思想是：某一指标在候选方案之间越离散，说明它越能区分方案优劣，信息效用越大，对应权重越高。具体地，先对每列指标作比例归一化得到 `p_ij`，再计算熵值 `e_j` 和信息效用 `d_j = 1 - e_j`，最后令 `w_j = d_j / sum(d_j)`。

当前运行得到的主方案权重如下：

| 层级 | 指标 | 熵权 |
|---|---|---:|
| 赛事影响 | `population_score` | 0.219924 |
| 赛事影响 | `gdp_score` | 0.360179 |
| 赛事影响 | `transport_score` | 0.192125 |
| 赛事影响 | `football_score` | 0.227773 |
| 承办能力 | `sports_base_score` | 0.091216 |
| 承办能力 | `stadium_capacity_score` | 0.383891 |
| 承办能力 | `rail_access_score` | 0.374049 |
| 承办能力 | `airport_access_score` | 0.150844 |
| 总目标 | `avg_travel_score` | 0.145361 |
| 总目标 | `worst_travel_score` | 0.680476 |
| 总目标 | `fairness_score` | 0.022698 |
| 总目标 | `impact_score` | 0.061259 |
| 总目标 | `capacity_score` | 0.090206 |

其中 `worst_travel_score` 权重最高，说明在所有候选的小组对和举办地组合中，最远出行负担的差异最明显，因而对方案优劣的区分作用最大。这与小组赛需要避免个别球队承担过高出行成本的公平性要求一致。

为避免单一权重方案决定结论，模型同时输出三套对照情景：

- `balanced`：综合均衡的专家权重方案。
- `travel_first`：更重视平均和最远旅行距离。
- `impact_first`：更重视人口、经济和传播影响力。

主方案为 `entropy`，其余三种方案用于敏感性分析。若不同情景下的核心赛点高度重合，可认为方案对权重扰动具有稳健性；若某些赛点发生替换，则在论文中将其解释为“出行便利”和“赛事影响”的权衡边界。

## 5. 求解流程

1. 读取问题 1 输出的 `results/problem1/best_groups.csv`。该文件是组员 A 在问题 1 中得到的最终分组结果，问题 3 的任务是在既定分组基础上选择比赛地点，因此直接继承该组数据作为输入，保证两个问题之间的决策链条一致。
2. 枚举 16 个小组中任意两个小组组成的组合。
3. 对每个小组组合，计算 64 个候选举办地的综合得分，并保留前若干个候选。
4. 使用受控 beam search 联合搜索 16 个小组的两两配对和 8 个互不重复举办地，使总得分尽量高。
5. 输出最终 8 个举办地、对应小组、关键评价指标、队伍距离明细、敏感性分析和台风图。

本文未采用 K-means 作为主求解方法，原因是 K-means 更适合连续空间聚类，而本题的举办地必须从 64 个离散候选地区中选择；同时，K-means 难以直接处理场馆容量、铁路与机场可达性、主场惩罚、小组两两配对和赛点唯一性等约束。相比之下，离散设施选址模型能更自然地表达本题约束，beam search 则用于在可控计算量内搜索高质量组合方案。

## 6. 结果解释口径

该模型不是简单挑选大城市，也不是先按地理位置聚类再就近找城市，而是让每个赛点服务一组具体参赛队伍。若某一县市交通和位置更适合某两个小组，即使综合影响力弱于省内核心城市，也可能被选为承办地。这样可以兼顾赛事传播、参赛便利和地域公平。

熵权主方案相对于综合均衡方案，更明显地压低了最远出行距离，但会牺牲一部分总影响力。这种结果不是模型错误，而是熵权法根据候选方案矩阵识别出的主要矛盾：在本题的数据结构中，最远出行负担最能区分不同赛点组合，因此主方案更偏向控制极端出行成本。
